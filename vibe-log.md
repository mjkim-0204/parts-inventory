## 2026-07-06 (회의 후) | 실제 팀 회의 결과 — 반출 기록 방식 변경 (개별 부품 단위 라벨)

### 배경
- 오늘 만든 PPT/노션 기반으로 팀 회의 진행. 발표 중 "스캔 방식"을 설명할 때 발표자 본인이 확신이 부족해서 버벅였고, 회의에서 더 논의한 결과 설계가 한 단계 더 나아감
- 기존 설계(Req 12/17)는 "부품 종류(SKU)당 QR 1장" + "시스템이 오래된 로트를 자동으로 제안"하는 방식이었음 — 이 경우 실제로 작업자가 다른 실물을 집어도 시스템은 확인할 방법이 없다는 한계가 있었음

### 결정 사항 (기존 설계에서 변경됨)
- **부품 낱개(실물) 하나하나에 고유 스티커/라벨을 붙이는 방식으로 확정** — 부품 종류당 1장이 아니라, 재고 수량만큼 라벨이 필요함 (예: 배수밸브 20개 재고면 라벨도 20장)
  - 번호 체계(시리얼 번호 vs 로트 번호)는 아직 미정, 추후 결정 필요
  - 마트 바코드(상품 종류당 공유 1개)보다 더 정밀한 방식 — 식품 유통업의 로트/유효기간 추적 방식에 가까움
- **스캔 기기: 무선 바코드 스캐너(SENSCAN 등) 대신 폰 카메라로 결정** — 별도 하드웨어 구매 계획 철회
- 개별 라벨 방식의 장점: 어떤 실물을 집었는지 스캔 순간 정확히 확인되므로, FIFO가 "시스템의 제안"이 아니라 "실제 확인된 사실"이 됨

### 영향받는 부분 (다음에 반영 필요)
- Req 12(로트/FIFO): 로트를 배치(입고 단위) 대신 개별 유닛 단위로 재설계할지 결정 필요
- Req 17(스캔 키오스크): 무선스캐너 → 폰 카메라 스캔 UI로 변경, "같은 QR 재스캔 시 수량+1" 로직도 "서로 다른 개별 코드를 부품종류별로 그룹핑해서 카운트"하는 방식으로 변경 필요
- 라벨 인쇄/발급 프로세스: 입고 시마다 들어온 개수만큼 신규 라벨을 발급하는 절차 설계 필요 (기존엔 부품종류당 1회성 라벨이었음)
- 하드웨어 준비물에서 무선 스캐너 항목 제거

### 다음 단계
- 번호체계(시리얼 vs 로트) 확정 후 requirements.md Req 12/17 개정
- design.md 작성 시 이 개별 유닛 단위 설계를 반영해서 진행

  - 15: 작업자/운영자 권한 분리 — Worker는 자기 차량 반출/반납만, Operator가 전체 관리
  - 16: Slack 알림 연동 — 최소재고 미달/발주서 생성 시 Slack Webhook 알림 (5분 배치)

- 발주는 "발주서 초안 생성"까지만 자동화, 실제 발주(전화/카톡)는 사람이 처리

## 2026-07-03 (계속) | 반출 스캔 키오스크(QR/바코드) 요구사항 확정

### 배경
- 지금 재고 추적이 안 맞는 근본 원인 논의: "총재고 - 선지출 = 재고" 구조인데, 작업자가 창고에서 여러 개를 가져가도 선지출은 실제 쓴 만큼만 기록해서 차이가 생김 → 반출(물리적 이동)과 선지출(청구/billing)을 분리해서 반출 기록을 선지출보다 먼저, 더 쉽게 만드는 게 핵심이라고 결론
- QR/바코드로 반출 기록을 쉽게 만드는 방법을 마트 셀프계산대 방식("카트 모드": 여러 개 스캔 후 한번에 확정)에 빗대어 설계
- 창고 규모(15평, 랙 3~4개), 부품 종류(~130개, 사진/이름 매칭 이미 완료, 구글설문지로 수집), 작업자 수(5명) 확인
- 하드웨어: 국내 브랜드 SENSCAN 핑거형 무선 2D 스캐너(블루투스 HID) + 안 쓰는 폰/태블릿 추천, 유선보다 무선이 창고 내 이동 워크플로우에 맞음
- 와이파이는 창고 형태(공유/단독) 미정 — 어느 쪽이든 Req 10의 오프라인 큐로 커버 가능하니 필수는 아님

### 결정 사항 → Requirement 17 "반출 스캔 키오스크" 신설
- 모드 토글(반출/반납/입고)을 먼저 선택 후 스캔
- 작업자 5명은 이름 탭 선택 (개인 QR/로그인 불필요, attribution 용도일 뿐 접근제어 아님 — Req 14의 기기 단위 인증과는 별개)
- 부품마다 Part_QR_Label(사진+이름 포함) 발급, 창고 선반/보관함에 부착용으로 A4 일괄 인쇄
- 스캔은 Cart_Session에 누적(같은 코드 재스캔 시 수량 자동 +1), 확정 시 일괄 처리
- QR 라벨 없는 신규/예외 부품은 수동 검색으로 대체 가능
- 오프라인 시 로컬 큐잉, 확정 전 세션 이탈 시 임시저장 후 재개

### 참고
- Requirement 12(FIFO)·Req 3(반출/반납)·Req 10(동기화)과 연동되도록 설계
